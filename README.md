# Streamflow Predictability Horizon Detection via SHAP Feature Shift

## Overview

This repository contains the analysis code for the manuscript:

> **"When Does a Streamflow Forecast Stop Being a Forecast? A SHAP-Based Framework for Detecting Predictability Horizon Collapse Across Contrasting Polish Catchments"**
> T. D. Muftuoglu (2026)

The study trains Optuna-optimised LightGBM and XGBoost models for five Polish gauging stations across five forecast lead times (h = 1, 2, 3, 5, 7 days) and uses SHAP (SHapley Additive exPlanations) at every lead time to track when a model transitions from genuine hydrological prediction to climatological anchoring. The lead time at which day-of-year seasonality harmonics first displace physical predictors at the top of the SHAP ranking is proposed as the **SHAP-based predictability horizon (SHAP-H)**.

## Key Results

| Station | h=1 R² | h=7 R² | SHAP-H |
|---|---|---|---|
| Nowy Targ | 0.644 | −0.098 | h = 3 |
| Nowy Sącz | 0.793 | −0.360 | h = 5 |
| Strekowa Góra | 0.996 | 0.887 | > h = 7 |
| Sieradz | 0.992 | 0.443 | > h = 7 |
| Sandomierz | 0.921 | 0.233 | h = 7 |

## Data Sources

**Discharge data:** Daily records were obtained from the [Global Runoff Data Centre (GRDC)](https://www.bafg.de/GRDC), Federal Institute of Hydrology, Koblenz, Germany. Data are available upon registration at the GRDC portal and are not redistributed here.

**Meteorological data:** Daily reanalysis data were downloaded from [NASA POWER](https://power.larc.nasa.gov) (Prediction Of Worldwide Energy Resources). The NASA POWER API allows free programmatic access; see `DATA_ACCESS.md` for exact download parameters used in this study.

## How to Run

The script is designed to run on **Google Colab** (free tier is sufficient; a GPU is not required).

### Step 1 — Upload input files

Open [Google Colab](https://colab.research.google.com), create a new notebook, and upload the 10 input files (5 GRDC + 5 NASA POWER) using the file panel on the left sidebar.

### Step 2 — Upload and run the script

Upload `streamflow_multibasin_final.py` to Colab and run:

```python
exec(open('streamflow_multibasin_final.py').read())
```

Or copy-paste the script contents into a Colab cell and run it directly.

### Step 3 — Download outputs

At the end of the run, all outputs are packaged into `multibasin_outputs_final.zip` and downloaded automatically. Total runtime is approximately 40–50 minutes on Colab free tier.
